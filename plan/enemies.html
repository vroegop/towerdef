Enemies

Two orthogonal axes — shape = behavior, color = strength tier — both defined as data, not code.

The two registries

Everything about an enemy is a lookup. A spawned enemy is just {type, tier} plus live state. The type decides how it moves and fights; the tier multiplies its numbers and picks its color.

// sim/enemies.js — TYPE registry: shape + behavior + base stats
export const TYPES = {
  melee:  { shape:'square',   behavior:'stick',  hp:20, dmg:6, speed:42, range:0,   r:11 },
  ranged: { shape:'triangle', behavior:'bounce', hp:14, dmg:5, speed:30, range:140, r:11 },
  boss:   { shape:'hexagon',  behavior:'stick',  hp:300,dmg:20,speed:20, range:0,   r:22 },
};

// TIER registry: color + stat multiplier + reward multiplier
export const TIERS = {
  weak:    { color:'#3ddc84', stat:0.6, reward:0.5 },  // green
  average: { color:'#4aa8ff', stat:1.0, reward:1.0 },  // blue
  hard:    { color:'#e64cff', stat:1.8, reward:2.2 },  // pink/purple
  elite:   { color:'#ffd24a', stat:3.0, reward:5.0 },  // gold
};

export function makeEnemy(type, tier, rng){
  const d = TYPES[type], t = TIERS[tier];
  return { type, tier, behavior:d.behavior, r:d.r,
    hp:d.hp*t.stat, hpMax:d.hp*t.stat, dmg:d.dmg*t.stat, speed:d.speed,
    range:d.range, x:0, y:0, vx:0, vy:0, facing:0, state:'approach' };
}
Adding a new enemy type = one entry in TYPES + one case in drawShape() + (optionally) one branch in the behavior switch. Adding a new rarity = one entry in TIERS. The spawner, balance, rewards, and renderer all read the registries, so nothing else needs touching.

Behaviors are a small switch, also data-keyed

function stepEnemy(e, hero, dt){
  const dx=hero.x-e.x, dy=hero.y-e.y, d=Math.hypot(dx,dy)||1;
  e.facing = Math.atan2(dy,dx);
  switch(e.behavior){
    case 'stick':   // melee: walk in, stay on hero unless out-sped
      if (d > hero.r+e.r) moveToward(e, dx,dy,d, dt);
      else e.state='stuck'; break;
    case 'bounce':  // ranged: hold at range, knocked back when hit
      if (d > e.range) moveToward(e, dx,dy,d, dt);
      else tryShoot(e, hero); break;
    // case 'orbit': ... case 'charger': ... add freely
  }
}
The "stick unless out-sped" rule falls out for free. Melee re-evaluates moveToward every tick. If the hero's speed > enemy speed, the hero simply opens distance faster than the enemy closes it, so it never reaches stuck. No special "can the hero escape?" code needed — it's emergent from the movement math.

Interactive · the whole art system in one function

Notice pentagon and diamond already render — they were added to drawShape() with a 2-line case each. That's the entire cost of a new enemy silhouette.

Wave timing & spawn caps

Implemented in sim/waves.js as pure functions of the wave number — tuning is editing numbers.

KnobValueMeaning
interval30sa new wave starts every 30 seconds
spawnWindow25sa wave's enemies all spawn within its first 25s
screenCap200concurrent cap — stays fixed
waveCount(n)min(140, 8+5·(n−1))how many a wave tries to send (the upgradable "spawn cap")
waveStr(n)1 + 0.08·(n−1)baseline strength multiplier per wave
waveSpeed(n)1 + 0.02·(n−1)baseline speed multiplier per wave
Spawn-cap vs screen-cap, resolved cleanly: a wave queues waveCount(n) releases over 25s, but each release only fires if alive < 200. So raising wave size (a future upgrade) only adds pressure when the arena isn't already full — exactly the rule you wanted, with no special-casing.

Enemy aging — the anti-hoarding pressure valve

When a new wave begins, every surviving enemy is strengthened. You chose the hybrid rule (in sim/enemies.js → ageSurvivors):

strMult = max( strMult × 1.1 ,  1.1 × (waveStr(n) × tierStat) )
speed   = max( speed   × 1.1 ,  1.1 × (baseSpeed   × waveSpeed(n)) )
  • Compounds ×1.1 for every wave an enemy survives — so a long-lived enemy becomes exponentially dangerous.
  • Floors at 1.1× a fresh same-tier enemy of the current wave — so survivors always out-class new spawns.
  • hp ratio is preserved on aging (a half-dead veteran stays half-dead, just with a bigger bar).

Consequence: kiting a giant ball of enemies forever turns it into a wall of veterans. Verified in a headless run — a survivor reached 4.2× strength by wave 6. The renderer draws a faint extra ring on aged enemies (veteran flag) so the threat is readable.
